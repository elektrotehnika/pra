# ПРА: Git - белешке са вежби

## Зашто постоји систем за верзионисање?

Систем за верзионисање (*Version Control System, VCS*) дозвољава вам да вратите датотеке на пређашње стање, да вратите читав пројекат на пређашње стање, да поредите измене током времена, да видите ко је последњи изменио нешто што би могло да буде узрок проблема који је настао, ко је објавио да постоји неки проблем и када, и још много тога. У општем случају, употреба VCS-а подразумева и то да ако нешто забрљате или изгубите датотеке, лако можете да их повратите.

## Зашто Git?

Git спада у дистрибуиране системе за верзионисање (DVCS) где корисници и сарадници на пројекту не преузимају само тренутан изглед датотека, већ потпуно пресликавају цео репозиторијум. Тако да ако неки од сервера са којих је пројекат преузет престане са радом, сваки од корисничких репозиторијума може да се ископира назад на сервер да би се он обновио. Сваки клон је буквално потпуна резервна копија свих података. 

Git је примарно направљен за потребе развоја Linux кернела, што значи да је направљен да рукује репозиторијумима са десетинама милиона линија кода од самог почетка. Брзина и перформансе су увек били примарни циљ у току развоја Git-a.

## Зашто GitHub?

Сервис [GitHub](https://github.com/) је највећи хост за Git репозиторијуме и представља централну тачку сарадње за милионе програмера и пројеката. Велики проценат свих Git репозиторијума се хостује на сервису GitHub и многи пројекти отвореног кода га користе за Git хостинг, праћење задатака, преглед кода, и друге ствари. Зато, иако није непосредни део Git пројекта отвореног кода, постоји велика вероватноћа да ћете у неком тренутку када будете почели професионално да користите програм Git бити упућени на интеракцију са сервисом GitHub.

<br>

Сада када имамо одговоре на питање *зашто*, хајде да разјаснимо и *како*!

## Неопходни ресурси 

Потребно је проверити да ли на вашем систему имате инсталиран `git`. На Linux-у то можемо проверити наредбом:

```
git --version
```

Ако на стандардном излазу видите исписану верзију, то значи да већ имате инсталиран `git`, што јесте случај на доста Linux дистрибуција након самог инсталирања система. Уколико ипак немате, преузмите га користећи [званично упутство](https://git-scm.com/install/linux) (на истој страници се налази и поступак за инсталацију на Windows оперативном систему). За потребе овог упутства коришћен је Linux оперативни систем.

Како бисте имали могућност повезивања удаљених (remote) репозиторијума и коришћења SSH протокола, неопходно је да [отворите GitHub налог и подесите SSH приступ](https://git-scm.com/book/en/v2/GitHub-Account-Setup-and-Configuration). Уколико имате потешкоћа у претходно наведеном поступку, обратите се за помоћ ономе ко вас је упутио на читање ових белешки.

Такође, на самом почетку, када већ имате инсталиран `git`, није сувишно поменути ни `help` опцију која је увек доступна из терминала:

```
git help
```

Или за нешто дужи списак доступних `git` наредби:

```
git help -a
```

док за конкретну `git` наредбу (команду) помоћ и списак доступних опција можете потражити на следећи начин:

```
git help <наредба>
```

## Git репозиторијум

Рецимо да најпре желимо да направимо развојно окружење за наш замишљени пројекат `pra_dev`. Први корак је прављење истоименог директоријума (нека се налази у `HOME` директоријуму) и прелазак у исти:

```
mkdir ~/pra_dev
cd ~/pra_dev
```

Унутар тренутног радног директоријума (`pra_dev`) извршити наредбу:

```
git init
```

Тиме је тренутни радни директоријум добио димензију локалног Git репозиторијума, а унутар њега се сада налази `.git` скривени поддиректоријум који ће носити све инфоримације о новонаправљеном Git репозиторијуму. Као и сви скривени директоријуми, може се излистати наредбом `ls -la`.

Тренутни статус Git репозиторијума се може добити наредбом:

```
git status
```

Испис би требало да буде следећи:

```
On branch master

No commits yet

nothing to commit (create/copy files and use "git add" to track)
```

Ако се до сада нисте сусрели са Git-ом, појмови као што су branch, commit, track су вам вероватно страни, али идемо редом!

Иницијализацијом Git репозиторијума наше развојно окружење постаје нешто више од обичног директоријума у датотечном систему. Сада је у оквиру тог директоријума доступна читава ризница `git` наредби која нам омогућава верзионисање његовог садржаја било да су у питању датотеке или поддиректоријуми.
